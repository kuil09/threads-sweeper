# Threads Sweeper

Threads 혐오 계정을 **대량으로 수집·차단**하는 Chrome / Chromium 계열 브라우저 확장 프로그램입니다.

## 소개

Threads에서 혐오 발언, 조롱, 극우 콘텐츠를 게시하는 계정들과 그 **팔로워 네트워크를 효율적으로 수집하고, 한 번에 차단**하는 도구입니다.  
브라우저에서 보이는 팔로워 목록을 자동으로 읽어와 **차단 대기열**에 쌓고, 백그라운드에서 여러 개의 작업창을 띄워 **동시에 차단**합니다.

## 주요 기능

### 1. 팔로워 수집 (스크롤 기반)
- 특정 유저의 프로필 페이지에서 **직접 팔로워 목록을 연 뒤**, 스크롤을 내리면 화면에 보이는 팔로워 계정들을 자동으로 인식합니다.
- 인식된 계정들은 사이드 패널의 **차단 대기열**에 실시간으로 추가됩니다.
- 동일 계정은 자동으로 중복 제거됩니다.

### 2. 차단 대기열 관리
- 수집된 계정은 사이드 패널의 리스트에서 확인할 수 있습니다.
- 입력창에 `username1 username2,username3` 처럼 공백/쉼표로 구분하여 **수동으로 계정 ID를 여러 개 추가**할 수 있습니다.
- 리스트는 `수집됨 / 차단 완료 / 실패` 상태로 표시되며, CSV로 내보낼 수 있습니다.

### 3. 백그라운드 대량 차단
- **[차단 시작]** 버튼을 누르면, 백그라운드 서비스 워커가 Threads 프로필 페이지를 자동으로 열어 **한 계정씩 차단 플로우를 실행**합니다.
- 확장 설정에서 **동시에 열어둘 작업창 개수(동시 차단 수)** 를 `1~10`개 사이에서 선택할 수 있어, 네트워크/PC 성능에 맞춰 속도를 조절할 수 있습니다.
- 각 계정의 차단 결과(성공/실패)가 대기열에 반영되며, 전체 진행 상황을 `현재/전체` 카운트로 확인할 수 있습니다.

### 4. CSV 내보내기
- 현재까지 **수집·처리된 계정 목록을 CSV 파일**로 저장할 수 있습니다.
- 열 구성: `Username, Status(collected|blocked|failed), Timestamp`

### 5. 작업 제어
- **수집 시작 / 중지**: 팔로워 수집 모드를 켜고 끌 수 있습니다.
- **차단 시작 / 일시 중지**: 차단 작업을 시작하거나 일시 정지할 수 있습니다. (일시 정지 시 큐는 유지됩니다.)
- **초기화**: 현재 대기열과 상태를 모두 비우고, 백그라운드 차단 작업을 중단합니다.

## 설치 방법

### 개발자 모드 설치 (권장)

1. 이 저장소를 클론하거나 ZIP으로 다운로드합니다:
   ```bash
   git clone https://github.com/your-username/threads-sweeper.git
   ```

2. Chrome 브라우저에서 `chrome://extensions` 로 이동합니다.

3. 우측 상단의 **개발자 모드**를 활성화합니다.

4. **압축해제된 확장 프로그램을 로드합니다** 버튼을 클릭합니다.

5. 다운로드한 `threads-sweeper` 폴더를 선택합니다.

6. 확장 프로그램이 설치되면 브라우저 툴바에 아이콘이 표시됩니다.

## 사용 방법

### 1. Threads 페이지에서 프로필 열기

1. Threads에서 차단하고자 하는 대상의 프로필 페이지로 이동합니다.
   - 예: `https://www.threads.net/@username`
2. 프로필에서 **팔로워 목록**을 직접 클릭해서 엽니다. (모달/패널 형태로 뜨는 팔로워 리스트)

### 2. 확장 프로그램 열기

1. 브라우저 툴바의 Threads Sweeper 아이콘을 클릭하면, 오른쪽에 **사이드 패널**이 열립니다.
2. 상단 카드에 현재 프로필의 `@username` 이 잡혀 있으면 준비 완료입니다.

### 3. 팔로워 수집하기

1. 사이드 패널 상단의 **[📥 수집 시작]** 버튼을 누릅니다.
2. Threads에서 열린 팔로워 목록을 **천천히 아래로 스크롤**합니다.
3. 화면에 보이는 팔로워 계정들이 자동으로 인식되어, 사이드 패널의 **차단 대기열** 리스트에 추가됩니다.
4. 수집을 멈추고 싶을 때는 **[⏸ 수집 중지]** 버튼을 다시 누르면 됩니다.

### 4. 차단 대기열 확인 및 수동 추가

1. 사이드 패널의 **입력창**에 `user1 user2,user3` 형식으로 계정 ID를 입력합니다. (`@`는 생략하거나 붙여도 상관없습니다.)
2. **[추가]** 버튼을 누르면 해당 계정들이 대기열에 합쳐집니다. (기존 수집분과 중복 제거)
3. **동시 차단** 드롭다운에서 한 번에 처리할 계정 수(동시 작업창 수)를 선택합니다.  
   - 예: `3개` → 최대 3개의 프로필 창을 띄워 병렬로 차단

### 5. 대량 차단 실행

1. **[차단 시작]** 버튼을 누르면, 백그라운드에서 Threads 프로필 창들을 띄우며 **자동으로 Block 버튼을 찾아 클릭**합니다.
2. 사이드 패널의 카운터가 `현재/전체` 형태로 증가하면서 진행률을 보여줍니다.
3. 잠시 멈추고 싶으면 **다시 [차단 시작] 버튼을 눌러 일시 정지**할 수 있습니다.
4. 모든 대기열이 비워지면, “모든 차단 작업이 완료되었습니다.” 메시지가 표시됩니다.

### 6. CSV로 결과 저장

1. 차단 결과를 별도로 보관하고 싶다면, **[CSV 저장]** 버튼(텍스트 링크)을 클릭합니다.
2. `threads_block_list_YYYY-MM-DD.csv` 형식의 파일이 다운로드됩니다.

> ⚠️ **중요**
> - 차단이 진행되는 동안에는 **브라우저를 종료하거나, 자동으로 뜬 Threads 창들을 닫지 않는 것**을 권장합니다.  
> - 너무 높은 동시 차단 수를 사용하면, Threads 측의 레이트 리밋 / 안전장치에 걸릴 가능성이 있습니다.

## 폴더 구조

```
threads-sweeper/
├── manifest.json              # Chrome Extension 설정 (Manifest V3)
├── src/
│   ├── background/
│   │   └── service-worker.js  # 백그라운드 서비스 워커 (차단 큐, 워커 풀, 윈도우 관리)
│   ├── content/
│   │   ├── content.js         # Threads 페이지 상호작용 스크립트 (프로필/팔로워 감지, 수집)
│   │   └── content.css        # 콘텐츠 스타일
│   ├── popup/
│   │   ├── popup.html         # 사이드 패널 UI
│   │   ├── popup.css          # 사이드 패널 스타일
│   │   └── popup.js           # 사이드 패널 로직 (대기열/상태 관리)
├── icons/                     # 확장 프로그램 아이콘 (SVG/PNG)
└── README.md
```

## 기술 스택

- **Chrome Extension Manifest V3**
- **웹 스크래핑 방식** (Threads 공식 API 미사용, DOM 기반 탐지)
- **HTML/CSS/JavaScript** - UI 및 로직
- **Service Worker + chrome.scripting** - 백그라운드 자동화

## 제약사항

- 자동 혐오 발언 탐지 기능 없음 (사용자가 직접 대상 선택)
- Threads 웹 구조 변경 시 유지보수 필요
- Threads의 이용약관 변경에 따라 기능이 제한될 수 있음
- 네트워크 상태/브라우저 상태에 따라 일부 계정 차단이 실패할 수 있으며, 이 경우 리스트에 `실패`로 표시됩니다.

## 주의사항

- 이 도구는 개인의 온라인 안전을 위한 방어적 목적으로 설계되었습니다.
- 차단 및 수집 과정에서 Threads 및 관련 서비스의 이용약관을 반드시 준수해야 합니다.
- 무분별한 차단은 피하고, 꼭 필요한 계정에만 사용해주세요.
- 법적 조치가 필요한 경우 전문 법률 상담을 받으시기 바랍니다.
